# Reports

This directory stores generated reports from validation, fuzzing, benchmarking,
and release-related checks. These files are committed so historical runs and
regressions are easy to audit.

What you will find:
- `test-orchestrator/`: full CI-style logs and summaries from `./tools/test-orchestrator`
- `fuzz/`: fuzzing artifacts and crash summaries
- `benchmarks/`: performance reports and comparisons
- `release-build/`: release build artifacts and logs
- `proof_pack*/`: proof pack outputs used for consistency checks

Notes:
- These files are generated by tooling and may change on each run.
- If you are looking for a specific failure, start with
  `test-orchestrator/*.log` and the summary at the end of each log.
